# MOTO STUNT — "진짜 게임" 완성 계획서 (2026-06-25, 검토용)

세 갈래 요청 정리: ① 16bit SFX ② VFX/아이템 부족분 ③ 전체 게임 완성도. 검토 후 진행.

---

## ① 16bit 사운드 (SFX/BGM) — 기술스택 확정

- **ZzFX** (KilledByAPixel) — 절차적 생성, **<1KB·파일 0개**, 20개 파라미터로 레트로 8/16비트 SFX. js13k 표준.
- **ZzFXM** — ZzFX 기반 초소형 음악 렌더러(패턴→스테레오 트랙). BGM 루프용.
- Howler.js(녹음파일 재생)는 불필요 — 우린 파일 없는 절차적 방식.
- 구현: `src/sfx.js` 모듈 — `sfx.play('name')`, `window.__muted` 연동(이미 메인화면 🔊 토글 있음), 엔진음은 속도 연동 루프.

### 필요 사운드 목록 (카테고리별 — 에이전트 분리 제작)
- **UI**: 버튼 hover/click · 메뉴 전환 · 카트선택 이동/확정(락) · 카운트다운 3·2·1·GO
- **엔진/주행**: 아이들 rev 루프(속도=피치) · 윌리 부스트 whoosh · 점프대 발사/착지
- **레이스**: 콘 충돌/전복 · 부활 · 피니시/우승 · 불꽃놀이
- **데스매치**: 트레일 사망 폭발 · 킬 확정음 · 경계 임박 경고 · 트레일 생성 틱(약하게)
- **아이템**: 지급 jingle · 점프/부스트/보호막/무적질주 발동 각각 · 점수 +/−
- **BGM**(선택): 메뉴 루프 1 · 인게임 루프 1 (ZzFXM)

### 제작 방식 (울트라코드 — 에이전트 분리)
- 카테고리별(UI/엔진/레이스/DM·아이템) 에이전트가 ZzFX 파라미터셋 설계 → 검수 → `sfx.js`에 통합 → 트리거 지점 배선.
- ⚠️ 에이전트는 소리를 못 들으니 파라미터는 합리적 추정. 통합 후 **유저가 듣고 튜닝**.

## ② VFX / 아이템 부족분

### 현재 있는 VFX
픽셀 디더 · 윌리 스파크 · 사망 폭발 · 불꽃놀이 · 트레일 발광 · 스포트라이트 · 고속 워프 · 플래시

### 채울 VFX (제안)
- **아이템 피드백**: 부스트=뒤 화염/속도라인 · 보호막=반투명 버블 · 무적질주=오라/잔상 · 점프=착지 먼지링
- **킬/점수 팝업**: 킬 시 "+2" 떠오름 · 피격 화면 가장자리 플래시
- **리스폰**: 등장 시 반짝임 + 무적 깜빡임(이미 일부)
- **점프대**: 발사 링 이펙트 · 착지 충격파
- **피니시**: 결승선 색종이(레이스)
- **트레일**: 사망 시 트레일 사라지는 디졸브

### 아이템 자체
- 현재 4종(점프/부스트/보호막/무적질주) 유지. **비주얼 피드백 추가가 핵심**. 추가 아이템 원하면 별도.
